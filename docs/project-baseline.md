## 概述
Grove 是一个基于 **事件驱动架构 (EDA)** 的智能研发协作助手，旨在模拟真实团队成员的自主协作行为。项目核心目标是通过集成 **GitHub**、**飞书 (Lark)** 及 **大语言模型 (LLM)**，实现从需求分析 (PRD)、任务拆解、代码评审到进度汇报的全流程自动化闭环。

**目标用户**：
- **研发团队**：需要自动化生成技术文档、智能分配任务及每日站会报告的开发者。
- **技术管理者**：希望通过数据驱动识别项目风险、监控进度漂移并减少重复沟通成本的项目经理或 Tech Lead。
- **DevOps 工程师**：寻求轻量级、可热插拔模块且易于容器化部署的自动化运维工具的用户。

## 技术架构
基于提供的仓库结构与源码，Grove 项目的技术架构分析如下：

**1. 技术栈**
*   **核心语言**：Python (基于 `pyproject.toml` 及 `.py` 文件推断)。
*   **Web 框架**：FastAPI (用于处理 GitHub/Lark Webhook 及管理接口)。
*   **配置管理**：Pydantic + YAML (强类型配置校验与环境变量注入)。
*   **外部集成**：GitHub App (异步/同步客户端)、飞书 (Lark) Open API (卡片消息、WebSocket)、LLM (DashScope/GLM)。
*   **部署运维**：Docker & Docker Compose，GitHub Actions (CI/CD)。
*   **数据存储**：目前主要为内存状态与文件系统 (`Storage` 类)，未见重型数据库依赖，倾向于轻量级持久化或外挂存储。

**2. 模块划分**
*   **`grove/core`**：基础设施层。包含事件总线 (`EventBus`)、模块注册中心、成员解析及统一存储接口，解耦业务逻辑。
*   **`grove/ingress`**：接入层。统一处理外部流量，包括 GitHub/Lark 的 Webhook、WebSocket 长连接、定时调度器 (`Scheduler`) 及管理端点。
*   **`grove/integrations`**：适配层。封装第三方 API 细节，提供标准化的 GitHub、Lark 及 LLM 客户端。
*   **`grove/modules`**：业务能力层。实现 11 个独立功能单元（如 `prd_generator`, `task_breakdown`, `daily_report` 等），每个模块通过事件驱动协作，具备独立的 Handler 与 Prompt 策略。

**3. 层次架构**
采用 **事件驱动架构 (EDA)** 结合 **模块化单体** 模式：
*   **核心机制**：所有组件通过 `EventBus` 进行异步通信，模块间无直接调用依赖，实现高内聚低耦合。
*   **流程模型**：`Ingress` 接收外部触发 -> 发布事件 -> `Core` 路由 -> `Modules` 消费并执行业务 (调用 LLM/外部 API) -> 发布新事件或回调通知。
*   **设计理念**：支持热插拔模块（通过配置开关），模拟真实团队成员的自主协作行为。
