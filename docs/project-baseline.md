## 概述
Grove 是一款面向现代研发团队智能化转型的**产品工程协作平台**。项目旨在解决传统研发流程中需求定义模糊、任务分配低效、进度同步滞后以及文档与代码脱节等痛点。

通过深度集成 **GitHub**（代码与Issue管理）与 **飞书 (Lark)**（即时通讯与文档），并结合 **大语言模型 (LLM)** 的推理能力，Grove 构建了一个事件驱动的自动化闭环系统。其核心目标用户为产品经理、技术负责人及开发团队，致力于实现从“需求对话”到"PRD生成”，再到“任务拆解分配”、“每日进度分析”及"PR评审同步”的全链路自动化，显著提升研发效能与信息透明度。

## 技术架构

### 1. 技术栈
*   **核心语言**：Python (基于 `pyproject.toml` 及源码结构推断)。
*   **Web 框架**：FastAPI (用于处理 GitHub/Lark Webhook 及管理接口)。
*   **异步编程**：`asyncio` (贯穿入口、客户端及事件总线，支持高并发 I/O)。
*   **配置管理**：Pydantic (强类型配置校验)、YAML。
*   **外部集成**：GitHub App API、飞书 (Lark) Open API (Bot/卡片/文档)、LLM (DashScope/GLM)。
*   **部署运维**：Docker & Docker Compose、GitHub Actions (CI)。
*   **数据存储**：目前主要依赖内存状态 (`Storage` 类) 及外部系统 (GitHub Issues/Lark Docs)，未见传统数据库依赖，偏向无状态或轻量级持久化。

### 2. 模块划分
*   **`grove/core`**：基础设施层。包含事件总线 (`EventBus`)、模块注册中心、成员解析及通用存储抽象，解耦业务逻辑。
*   **`grove/ingress`**：接入层。统一处理外部流量，包括 GitHub/Lark 的 Webhook、WebSocket 长连接、定时调度器 (`scheduler`) 及管理端点。
*   **`grove/integrations`**：适配层。封装第三方 SDK，提供 GitHub、Lark 及 LLM 的标准化异步客户端。
*   **`grove/modules`**：业务能力层。包含 11 个独立功能模块（如 `prd_generator`, `task_breakdown`, `daily_report` 等），每个模块通过事件总线交互，实现具体产品管理职能。
*   **`docs`**：规划与规范。包含分阶段的产品路线图 (Phase 1-6) 及架构设计文档。

### 3. 层次架构
项目采用 **事件驱动架构 (Event-Driven Architecture)** 结合 **模块化单体** 模式：
*   **核心机制**：所有模块通过 `EventBus` 进行松耦合通信。ingress 层接收外部事件后发布到总线，各业务模块订阅相关事件并异步处理。
*   **分层逻辑**：遵循“接入 (Ingress) -> 核心调度 (Core/EventBus) -> 业务执行 (Modules) -> 外部交互 (Integrations)"的流向。
*   **特点**：具备高度可扩展性，新功能只需注册新模块并监听事件即可，无需修改核心流程；支持热插拔配置 (`ModulesConfig`) 动态启停功能。
