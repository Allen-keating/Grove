## 概述
Grove 是一款面向现代敏捷开发团队的**AI 驱动型产品管理自动化平台**。该项目旨在解决产品研发过程中需求文档（PRD）维护滞后、任务分配不均、进度同步繁琐以及跨平台（GitHub 代码库与飞书协作平台）信息割裂等痛点。

通过集成大语言模型（LLM）智能引擎，Grove 能够自动解析代码变更、生成与更新产品文档、智能分解任务并分配给合适成员，同时提供每日站会报告与风险预警。其核心目标用户为**产品经理、技术负责人及研发团队成员**，帮助他们从重复的文档维护和沟通协调中解放出来，专注于核心业务逻辑与创新。项目采用事件驱动架构，确保系统在高并发场景下的稳定性与模块扩展的灵活性。

## 技术架构

### 1. 技术栈
*   **核心语言**：Python (基于 `pyproject.toml` 及源码结构推断)。
*   **Web 框架**：FastAPI (用于处理 GitHub/Lark Webhook 及管理接口)。
*   **异步编程**：Asyncio (贯穿客户端与调度器，支持高并发 I/O)。
*   **配置管理**：Pydantic + YAML (强类型配置校验与环境变量注入)。
*   **外部集成**：GitHub App (Git 操作/Issue 管理)、飞书/Lark Open API (IM 交互/卡片消息/云文档)、LLM (DashScope/GLM 大模型接口)。
*   **部署运维**：Docker & Docker Compose。
*   **数据存储**：轻量级本地存储 (`grove/core/storage.py`，可能为 JSON/SQLite)，无重型数据库依赖。

### 2. 模块划分
*   **`grove/core`**：基础设施层。包含事件总线 (`event_bus`)、模块注册中心、成员解析及统一存储接口，解耦业务逻辑。
*   **`grove/ingress`**：入口适配层。封装 GitHub/Lark 的 Webhook、WebSocket 连接及定时调度器 (`scheduler`)，负责流量接入。
*   **`grove/integrations`**：外部服务客户端。封装 GitHub API、Lark SDK 及 LLM 调用逻辑，提供统一异步客户端。
*   **`grove/modules`**：业务能力层。包含 11 个独立功能模块（如 `prd_generator`, `task_breakdown`, `daily_report` 等），每个模块通过事件总线订阅/发布消息，实现具体 AI 产品管理职能。

### 3. 层次架构
项目采用 **事件驱动架构 (Event-Driven Architecture)** 结合 **模块化单体 (Modular Monolith)** 模式：
*   **核心机制**：所有组件通过 `EventBus` 进行松耦合通信。入口层将外部触发（如 Webhook、定时任务）转化为标准事件，业务模块监听特定事件执行逻辑并产生新事件。
*   **分层设计**：
    *   **接入层** (`ingress`) 负责协议转换。
    *   **核心层** (`core`) 提供运行时支撑。
    *   **适配层** (`integrations`) 屏蔽第三方差异。
    *   **领域层** (`modules`) 专注业务规则与 AI 提示词工程。
*   **特点**：高内聚低耦合，支持通过配置文件动态启用/禁用模块，便于扩展新功能而不影响主干流程。
