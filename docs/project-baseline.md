## 概述
Grove 是一款面向现代软件研发团队的**智能产品管理助手**。它旨在通过事件驱动架构，深度融合 GitHub 代码协作平台与飞书（Lark）即时通讯工具，利用大语言模型（LLM）能力自动化执行产品需求文档（PRD）生成、任务拆解、进度追踪及风险预警等核心职能。

本项目主要服务于**产品经理、技术负责人及研发团队**，目标是减少重复性沟通成本，确保需求与代码实现的一致性，并通过数据驱动的每日报告提升团队透明度与交付效率。系统采用模块化单体设计，支持功能的动态热插拔，兼具部署的便捷性与业务扩展的灵活性。

## 技术架构

### 1. 技术栈
*   **核心语言**：Python (基于 `pyproject.toml` 及源码结构推断)。
*   **Web 框架**：FastAPI (用于处理 GitHub/Lark Webhook 及管理接口)。
*   **异步编程**：Asyncio (贯穿客户端与调度器，支持高并发 I/O)。
*   **配置管理**：Pydantic + YAML (强类型配置校验与环境变量注入)。
*   **外部集成**：GitHub App (Git 操作)、飞书/Lark Open Platform (IM 卡片、文档、机器人)、LLM (DashScope/GLM)。
*   **部署方式**：Docker & Docker Compose。
*   **数据存储**：目前主要为内存状态或轻量级文件存储 (`core/storage.py`)，未见重型数据库依赖。

### 2. 模块划分
*   **`grove/core`**：基础设施层。包含事件总线 (`event_bus`)、模块注册表、成员解析及统一存储接口，解耦业务逻辑。
*   **`grove/ingress`**：接入层。统一处理外部流量，包括 GitHub/Lark 的 Webhook、WebSocket 长连接、定时调度器 (`scheduler`) 及管理 API。
*   **`grove/integrations`**：适配层。封装第三方 SDK，提供 GitHub、Lark 及 LLM 的标准化异步客户端。
*   **`grove/modules`**：业务能力层。包含 11 个独立功能模块（如 `prd_generator`, `task_breakdown`, `daily_report` 等），每个模块通过事件总线交互，实现具体 AI 产品管理职能。

### 3. 层次架构
项目采用 **事件驱动架构 (Event-Driven Architecture)** 结合 **模块化单体** 模式：
*   **通信机制**：所有组件通过中央 `EventBus` 进行松耦合通信，入站请求转化为内部事件，由对应模块消费处理。
*   **分层设计**：遵循“接入层 (Ingress) -> 核心协调层 (Core) -> 业务模块层 (Modules) -> 集成适配层 (Integrations)"的清晰分层。
*   **运行模式**：单进程异步运行，通过配置动态加载/卸载功能模块，兼具微服务的灵活性与单体的部署便捷性。
